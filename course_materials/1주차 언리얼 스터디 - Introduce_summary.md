## 언리얼을 선택하는 이유

사람들이 언리얼을 배우려는 흔한 이유:

1. **그래픽이 예쁘다** — 단, 실제 게임은 CG가 아니며 유저 PC는 하이엔드가 아님. 퍼포먼스 최적화가 필수.
2. **기능이 많다** — 업데이트 속도가 빨라 공식 문서가 따라가지 못하는 경우 多. 블로그/AI만으로는 한계가 있어 소스코드 독해 능력이 중요.
3. **언리얼로 만든 게임을 해봤다** — 업계 레퍼런스가 풍부.
4. **채용이 많다** — 다수 회사가 언리얼 기반 개발자를 채용 중이며, C++ 역량을 주요 자격 요건으로 요구.

## 언리얼 엔진의 특징

### 엄격한 프레임워크

- 유니티: `MonoBehaviour`를 자유롭게 배치하는 구조
- 언리얼: 시작부터 `GameMode`, `PlayerController`, `GameState` 등 매니저가 스폰되는 강제 구조
- 제약이 많지만, 그 덕분에 대형 프로젝트에서 구조의 일관성이 보장된다.

### C++ Native

| C++ 특징 | 모던 객체지향 특징 |
|---|---|
| 탁월한 메모리 관리 | 메모리 관리 자동화 |
| 높은 객체 설계 자유도 | 상속보다 합성 |
| 하드웨어 친화적 | 인터페이스 기반 설계 |
| | 리플렉션을 통한 객체 관리 |

언리얼 C++는 두 방식의 교집합에 위치한다 — C++의 하드웨어 접근성을 유지하면서 모던 객체지향 패턴을 적용.

**모던 객체지향 실전 예시:** 캐릭터를 역할별 컴포넌트로 분리 (`CapsuleComponent`, `CharacterMovement`, `CameraRoot`, `Mesh` 등), 모델링만 다른 캐릭터나 AI 캐릭터로의 확장이 용이.

### Blueprint

- C++ 코드와 동등한 기능을 노드 기반 비주얼 스크립팅으로 구현
- 빠른 컴파일 및 결과 확인, 기획자 친화적
- C++ 구현체와 Blueprint 구현체는 **동일한 결과**를 낸다

### 아티스트 친화적 생태계

- 3D 모델링 툴 연동
- 고성능 렌더링 파이프라인
- 영상·게임 외 영역(K-pop 뮤직비디오 프리비즈 등)에서도 활용

### 오픈소스

- 엔진은 블랙박스가 아니다 — 소스코드를 직접 열어볼 수 있다.
- `AGameMode`, `AGameModeBase`, `AInfo` 등 엔진 클래스의 설계 의도를 코드 수준에서 확인 가능.

## 언리얼을 배우려는 사람에게 (강사 조언)

1. **좋은 소스코드를 많이 보라** — Unreal 공식 샘플 프로젝트 활용
2. **엔진을 블랙박스로 보지 마라** — 내부 동작 원리가 공개된 API는 드물다. 적극적으로 소스를 파헤쳐라.
3. **무엇이든 하나 직접 만들어 보라** — 어떤 강의도 직접 만드는 경험보다 많이 가르쳐주지 못한다.
