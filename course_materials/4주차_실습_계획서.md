## 실습 0 — Possess 체험 (10분)

### 목표
`APlayerController::Possess()`를 직접 호출해, 코드 변경 없이 원하는 폰으로 빙의하는 흐름을 체험한다.

### 만들 것
| 에셋/클래스 | 종류 | 설명 |
|---|---|---|
| `AWeek3PossessSelector` | C++ Actor | 어느 폰에 빙의할지 지정하는 Selector |
| `AWeek3PlayerController` | C++ PlayerController | BeginPlay에서 Selector를 찾아 Possess 호출 |
| `BP_Week3PlayerController` | Blueprint child | PlayerController의 블루프린트 래퍼 |

### 레벨 준비
1. `BP_ThirdPersonCharacter` 두 개를 레벨에 배치 → 이름: `CharacterA`, `CharacterB`
2. World Settings → GameMode Override → `BP_Gamemode`
3. `BP_Gamemode` → Default Pawn Class: **None** (자동 스폰 비활성)

### Step 1 — Week3PossessSelector 헤더

**파일:** `Source/Week3/Week3PossessSelector.h`

```cpp
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "Week3PossessSelector.generated.h"

UCLASS()
class WEEK3_API AWeek3PossessSelector : public AActor
{
    GENERATED_BODY()

public:
    // 레벨 디테일 패널에서 교체 가능
    UPROPERTY(EditAnywhere, Category="Possess")
    APawn* CharacterToControl;
};
```

### Step 2 — Week3PlayerController 헤더

**파일:** `Source/Week3/Week3PlayerController.h`

```cpp
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/PlayerController.h"
#include "Week3PlayerController.generated.h"

UCLASS()
class WEEK3_API AWeek3PlayerController : public APlayerController
{
    GENERATED_BODY()

protected:
    virtual void BeginPlay() override;
};
```

### Step 3 — Week3PlayerController 구현

**파일:** `Source/Week3/Week3PlayerController.cpp`

```cpp
#include "Week3PlayerController.h"
#include "Week3PossessSelector.h"
#include "EngineUtils.h"   // TActorIterator

void AWeek3PlayerController::BeginPlay()
{
    Super::BeginPlay();

    // 레벨에서 Selector 탐색
    for (TActorIterator<AWeek3PossessSelector> It(GetWorld()); It; ++It)
    {
        AWeek3PossessSelector* Selector = *It;
        if (Selector && Selector->CharacterToControl)
        {
            Possess(Selector->CharacterToControl);
        }
        break; // 첫 번째 Selector만 사용
    }
}
```

### 체크포인트
- PIE 시작 → CharacterToControl에 지정한 폰으로 빙의됨
- 디테일 패널에서 CharacterB로 변경 → 다음 PIE에 B로 빙의됨
- `Week3Character` 코드를 한 줄도 수정하지 않음

## 실습 1 — 스프린트 구현 (15분)

### 목표
`IA_Sprint` 액션을 생성하고, `Shift` 키를 누르는 동안 캐릭터 이동 속도를 높인다.

### 만들 것
| 에셋/코드 | 종류 | 설명 |
|---|---|---|
| `IA_Sprint` | InputAction | Digital(bool) 타입의 스프린트 액션 |
| `IMC_Default` 매핑 추가 | - | Left Shift → IA_Sprint |
| `Sprint()` / `StopSprint()` | C++ 멤버 함수 | MaxWalkSpeed 변경 |

### Step 1 — IA_Sprint 에셋 생성

Content Browser → 우클릭 → Input → **Input Action**  
- 이름: `IA_Sprint`  
- Value Type: `Digital (bool)`

### Step 2 — IMC_Default 매핑 추가

`IMC_Default` 열기 → Mappings + →  
- Action: `IA_Sprint`  
- 키: `Left Shift`

### Step 3 — Week3Character.h 변경사항

**파일:** `Source/Week3/Week3Character.h`

```cpp
// 기존 InputAction 변수들 아래에 추가
UPROPERTY(EditAnywhere, BlueprintReadOnly, Category="Input")
UInputAction* SprintAction;

UPROPERTY(EditAnywhere, Category="Movement")
float SprintSpeed = 800.f;

UPROPERTY(EditAnywhere, Category="Movement")
float WalkSpeed = 500.f;

// 함수 선언
void Sprint(const FInputActionValue& Value);
void StopSprint(const FInputActionValue& Value);
```

### Step 4 — Week3Character.cpp 구현

**파일:** `Source/Week3/Week3Character.cpp`

```cpp
void AWeek3Character::Sprint(const FInputActionValue& Value)
{
    GetCharacterMovement()->MaxWalkSpeed = SprintSpeed;
}

void AWeek3Character::StopSprint(const FInputActionValue& Value)
{
    GetCharacterMovement()->MaxWalkSpeed = WalkSpeed;
}
```

**SetupPlayerInputComponent 내 바인딩 추가:**

```cpp
// 기존 바인딩 코드 아래에 추가
if (SprintAction)
{
    EnhancedInputComponent->BindAction(
        SprintAction, ETriggerEvent::Started,   this, &AWeek3Character::Sprint);
    EnhancedInputComponent->BindAction(
        SprintAction, ETriggerEvent::Completed, this, &AWeek3Character::StopSprint);
}
```

> `Started` — 키를 처음 누르는 순간 1회  
> `Completed` — 키를 떼는 순간 1회

### 블루프린트 연결
빌드 후 `BP_ThirdPersonCharacter` 디테일 패널 → **SprintAction** 슬롯에 `IA_Sprint` 에셋 연결

### 체크포인트
- Shift 누르는 동안 이동 속도 증가
- Shift 떼면 원래 속도 복귀
- BP 디테일에서 SprintSpeed 값 변경 가능

## 실습 2 — HP 바 UMG 위젯 (20분)

### 목표
`WBP_PlayerHUD` 위젯을 만들고, `PlayerState`의 HP를 실시간으로 반영하는 ProgressBar를 화면에 표시한다.

### 만들 것
| 에셋 | 종류 | 설명 |
|---|---|---|
| `WBP_PlayerHUD` | Widget Blueprint | HP 바 UI |
| `PlayerController::BeginPlay` 수정 | Blueprint | 위젯 생성 + 뷰포트 추가 |

### Step 1 — 위젯 블루프린트 생성

Content Browser → 우클릭 → User Interface → **Widget Blueprint**  
- 이름: `WBP_PlayerHUD`  
- 기본 클래스: `UserWidget`

### Step 2 — UI 레이아웃

```
Canvas Panel (루트)
└── Vertical Box (좌하단 앵커)
    ├── Text Block: "HP"
    └── Progress Bar
        ├── Size: 200 × 20
        ├── Fill Color: R=1, G=0, B=0 (빨간색)
        └── Is Variable: ✅ 체크 → 변수명: HPBar
```

### Step 3 — Percent 바인딩

`HPBar` → Details → Percent → **Bind** → Create Binding

바인딩 함수 로직:
```
[Get Owning Player] → [Get Player State] → [Cast to BP_PlayerState]
    → [Get CurrentHP] ÷ [Get MaxHP] → [Return Node (Float)]
```

> **주의:** Cast 실패 분기에서 `0.0`을 반환하도록 처리해야 크래시 방지

### Step 4 — PlayerController에서 위젯 생성

`BP_Week3PlayerController` → **Event BeginPlay**:

```
[Create Widget (WBP_PlayerHUD)] → [Add to Viewport]
                                → [변수 HUDWidget에 저장]
```

### 체크포인트
- 게임 시작 시 좌하단에 빨간 프로그레스 바 표시
- Percent = 1.0 (100%로 꽉 참)
- PIE 재시작 후에도 표시됨

## 실습 3 — PlayerState HP + 데미지 처리 (20분)

### 목표
`BP_PlayerState`에 HP 변수와 `TakeDamage` 함수를 구현하고, `G` 키로 데미지를 주어 HP 바가 줄어드는 것을 확인한다.

### 만들 것
| 에셋/코드 | 종류 | 설명 |
|---|---|---|
| `BP_PlayerState` | Blueprint PlayerState | MaxHP, CurrentHP, TakeDamage, IsAlive |
| `IA_DebugDamage` | InputAction | G 키 → 10 데미지 |
| `DebugDamage()` | C++ 멤버 함수 | PlayerState.TakeDamage(10) 호출 |

### Step 1 — BP_PlayerState 생성

Content Browser → Blueprint Class → **PlayerState** 검색 → 선택  
- 이름: `BP_PlayerState`

**변수 추가:**
| 이름 | 타입 | 기본값 |
|---|---|---|
| `MaxHP` | Float | 100.0 |
| `CurrentHP` | Float | 100.0 |

**Event BeginPlay:** `CurrentHP = MaxHP`

**World Settings → GameMode → Player State Class: `BP_PlayerState`**

### Step 2 — TakeDamage / IsAlive 함수

`BP_PlayerState` → My Blueprint → 함수 추가

**TakeDamage(DamageAmount: Float):**
```
CurrentHP = Clamp(CurrentHP - DamageAmount, 0.0, MaxHP)
```

**IsAlive() → Boolean:**
```
Return (CurrentHP > 0.0)
```

### Step 3 — IA_DebugDamage 생성 및 매핑

- `IA_DebugDamage` 생성 → Value Type: `Digital (bool)`
- `IMC_Default` → Mappings 추가: `G` 키 → `IA_DebugDamage`

### Step 4 — Week3Character.h 변경사항

```cpp
UPROPERTY(EditAnywhere, BlueprintReadOnly, Category="Input")
UInputAction* DebugDamageAction;

void DebugDamage(const FInputActionValue& Value);
```

### Step 5 — Week3Character.cpp 구현

```cpp
#include "GameFramework/PlayerState.h"

void AWeek3Character::DebugDamage(const FInputActionValue& Value)
{
    APlayerState* PS = GetPlayerState();
    if (!PS) return;

    // BP_PlayerState는 블루프린트 클래스이므로 런타임 캐스트 사용
    // 실제 프로젝트에서는 C++ PlayerState를 만들어 직접 캐스팅 권장
    UFunction* TakeDamageFunc = PS->FindFunction(FName("TakeDamage"));
    if (TakeDamageFunc)
    {
        struct { float DamageAmount; } Params{ 10.f };
        PS->ProcessEvent(TakeDamageFunc, &Params);
    }
}
```

> **대안 (더 안전한 방법):** C++ `AWeek3PlayerState`를 만들고 `TakeDamage`를 C++로 선언  
> 블루프린트 함수를 C++에서 직접 캐스팅하려면 블루프린트 생성 클래스명(`ABP_PlayerState_C`)이 필요해 깨지기 쉬움

**SetupPlayerInputComponent 바인딩 추가:**
```cpp
if (DebugDamageAction)
{
    EnhancedInputComponent->BindAction(
        DebugDamageAction, ETriggerEvent::Started, this, &AWeek3Character::DebugDamage);
}
```

**블루프린트 연결:** `BP_ThirdPersonCharacter` 디테일 → `DebugDamageAction` 슬롯에 `IA_DebugDamage` 연결

### 체크포인트 — 전체 흐름 완성
- G 키 → IA_DebugDamage → Character → PlayerState.TakeDamage(10)
- HP 바가 10씩 줄어드는 것이 화면에 보임
- HP = 0 → IsAlive() = false
- 전체 데이터 흐름: **Input → Controller → Pawn → PlayerState → UI**
