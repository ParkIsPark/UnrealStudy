### 핵심 인사이트

1. **이론 → 실습 전환 요구**: 강사가 2주차에 "이론만으로는 흥미 유발이 어렵다"고 판단, 3주차부터 실습 비중 확대
2. **진도 밀림**: 원본 1주차의 '기본 문법/상속'과 3주차의 'PlayerCharacter/PlayerState'가 아직 다뤄지지 않음
3. **수강생 환경 편차**: 3주차 실습 시 엔진 설치 상태 점검에 시간 소요 → 사전 환경 구축 확인 필요

### 미진행 주제 (원본 1~3주차 중)

- [ ] C++ 기본 문법 및 상속 (언리얼 맥락)

## 2. 수정 강의 계획

### 설계 원칙

- **매주 이론 30~40분 + 실습 50~60분** 구성으로 전환 (원본은 이론 위주)
- 미진행 주제를 남은 수업에 자연스럽게 편입
- **실습 연계 프로젝트**: 매주 배운 내용을 하나의 미니 게임 프로젝트에 누적 적용하여 체감 효과 극대화
- 원본의 에셋 관리 2개 주차를 1개로 압축하여 프로젝트 마무리 시간 확보

---

### 주차별 계획

#### 4주차 — 4/27 | PlayerController, PlayerState + HUD 기초

**이론 (30분)**
- 미진행 주제 보충: PlayerCharacter / PlayerState 역할과 구조
- PlayerController: 입력 처리와 Pawn 빙의(Possess) 메커니즘
- HUD 개요: UMG(Unreal Motion Graphics) 아키텍처

**실습 (60분)**
- Enhanced Input System으로 캐릭터 이동/점프 구현
- 간단한 HUD 위젯(HP 바) 블루프린트로 제작
- PlayerState에 HP 변수 추가 → HUD에 바인딩

**비고**: 3주차 미진행분(PlayerCharacter/PlayerState) 보충 + 원본 4/27 HUD 내용 통합

---

#### 5주차 — 5/4 | 블루프린트 기초 + 언리얼 C++ 문법

**이론 (30분)**
- 미진행 주제 보충: 언리얼식 C++ 문법 — UPROPERTY, UFUNCTION 매크로
- 가비지 컬렉션 기본 원리
- 리플렉션 시스템이 블루프린트 연동을 가능하게 하는 구조

**실습 (60분)**
- C++ 클래스 생성 → UPROPERTY로 에디터 노출
- 같은 기능을 블루프린트로 구현하여 C++/BP 비교 체험
- 1주차 미진행분(상속) 보충: ACharacter 상속 커스텀 클래스 작성

**비고**: 원본 5/4 (C++ 심화) + 원본 4/27 블루프린트 기초 + 1주차 미진행 문법/상속 통합

---

#### 6주차 — 5/11 | 델리게이트와 컴포넌트 설계 + 오브젝트 간 통신

**이론 (30분)**
- 델리게이트(Delegate): 싱글캐스트, 멀티캐스트, 다이나믹
- 컴포넌트 기반 설계: 기능을 컴포넌트로 분리하는 이유와 방법
- 라이프사이클 보충: BeginPlay, Tick, EndPlay 실행 순서

**실습 (60분)**
- 데미지 시스템 구현: 델리게이트로 HP 변경 이벤트 → HUD 자동 업데이트
- ActorComponent 커스텀 제작 (예: HealthComponent)
- 오브젝트 간 통신 실습: 트리거 볼륨 → 델리게이트 → 반응

**비고**: 원본 5/11 내용 유지 + 라이프사이클 미진행분 보충

---

#### 7주차 — 5/18 | 에셋 관리 — 포인터, DataAsset, Data-Driven 설계

**이론 (40분)**
- 원본 5/18 + 5/25 통합: TObjectPtr, TSoftObjectPtr, UClass 참조 방식
- DataAsset: 데이터 중심 설계의 핵심 도구
- PrimaryAsset 간략 소개 (대규모 프로젝트용)

**실습 (50분)**
- DataAsset으로 아이템/무기 데이터 테이블 작성
- SoftPtr로 에셋 비동기 로딩 체험
- 기존 프로젝트에 데이터 기반 아이템 시스템 적용

**비고**: 원본 2주 분량을 1주로 압축. PrimaryAsset은 개념 소개만 (실무 심화는 별도)

---

#### 8주차 — 5/25 | 코딩 컨벤션 + 프로젝트 종합 실습

**이론 (20분)**
- 언리얼 코딩 컨벤션: 네이밍 규칙, 헤더 구성, 모듈 구조
- 프로젝트 구조 설계 가이드: 폴더 구성, 에셋 네이밍

**실습 (70분)**
- 지금까지 만든 기능들을 하나의 미니 게임으로 통합
- 코드 리뷰: 컨벤션에 맞게 리팩토링
- 팀 프로젝트 또는 개인 프로젝트 방향 논의

**비고**: 원본 5/4의 코딩 컨벤션 부분을 여기로 이동. 마무리 + 확장 방향 제시

## 4. 권장 사항

1. **사전 환경 체크**: 중간고사 기간(4/13~4/20)에 수강생 전원 엔진 버전·프로젝트 세팅 사전 확인 공지 → 4/27 실습 시간 낭비 방지
2. **실습 프로젝트 연속성**: 4주차부터 매주 동일 프로젝트에 기능을 추가하는 방식으로 진행하면 학습 내용이 누적되어 성취감 극대화
3. **보충 자료 제공**: 1주차에 다루지 못한 C++ 기본 문법/상속은 사전 읽기 자료로 배포하여 5/4 실습 전까지 자습 유도
4. **PlayerState 깊이 조절**: 3주차 미진행분인 PlayerState는 4/27에 다루되, 멀티플레이어 Replication은 범위 밖으로 설정 (시간 제약)
5. **유연한 8주차 활용**: 수강생 진도에 따라 8주차를 미진행 주제 보충 또는 프로젝트 발표로 전환 가능
