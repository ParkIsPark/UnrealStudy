## 2주차 — GameMode / GameState (2026-03-30)

**형태**: 전체 이론 수업

### 다룬 핵심 내용

**문제 제기 — 게임 요소 관리의 어려움**
- 수많은 게임 요소(Sound, Network, Character, Skill 등)의 생성/소멸 타이밍 불일치
- God Object 안티패턴: 하나의 GameManager에 모든 것을 넣으면 객체지향 설계 파괴
- 프로젝트마다 GameManager를 재작성하는 비효율
- **언리얼의 철학**: "모두가 통일된 프레임워크로!"

**월드 생성 순서**
```
엔진 실행 → GameInstance → UWorld → World Component → GameMode(최우선 스폰) → GameState 등 → 게임 시작
```

**GameMode (룰 담당)**
- 게임 내 오브젝트 중 가장 먼저 스폰
- 서버에서만 존재 (클라이언트에 없음)
- 담당: 게임 룰, 승/패 판단, 플레이어 접속 관리

**GameState (상태 담당)**
- 서버 + 모든 클라이언트에 복제 (Replication)
- 담당: 게임 진행 상태, 플레이어 목록, 서버 시간 등

**멀티플레이어 설계 원칙**
- 룰(GameMode): 서버만 보유
- 상태(GameState): 전체 공유 (복제)

### 실습 수업 전환 예고

> 2주차는 전체 이론 수업으로 진행되었으며, 강사 스스로 "조금 재미없을 수도 있는 내용"이라 언급.  
> **"이론 수업만으로는 흥미 유발과 실제 체험 제공이 어렵다"는 인식 하에, 3주차부터 실습 수업 비중을 늘리겠다고 예고.**  
> 이 이론적 기반이 이후 실습 진행 시 막힘 없는 진행의 선행 조건으로 제시됨.

## 3주차 — UObject / AActor / Pawn / Character (2026-04-06)

**형태**: 이론 + 실습 혼합

### 다룬 핵심 내용

**UObject — 언리얼의 최상위 기반 클래스**
- 메모리 상에 존재하는 모든 데이터의 기반
- 제공 기능: GC, 레퍼런스 업데이트, 리플렉션, 직렬화, 에디터 통합, 네트워크 리플리케이션
- 월드에 종속되지 않음 (엔진 전반에서 사용)

**AActor — 월드(Level)에 종속된 오브젝트**
- UObject를 상속, Transform(위치/회전/스케일) 보유
- ULevel의 `Actors` 배열에 등록되어 관리
- "Scene 내 모든 것은 AActor, AActor를 포함한 모든 것은 UObject"

**APawn — 빙의(Possess) 가능한 Actor**
- 플레이어 또는 AI가 입력을 통해 조종 가능
- 단순 좌표 이동만 구현 (물리 미적용)

**ACharacter — 인간형 캐릭터 특화 클래스**
- Pawn을 상속
- 기본 컴포넌트: CapsuleComponent, SkeletalMesh, CameraComponent, CharacterMovementComponent(CMC), PerformerComponent
- CMC: 물리 기반의 자연스러운 이동 구현, 언리얼 엔진의 대표적인 잘 설계된 컴포넌트

**클래스 계층 구조**
```
UObject
  └─ AActor (위치 보유, World 소유)
       └─ APawn (플레이어 입력 가능)
            └─ ACharacter (인간형, CMC 포함)
```

**실습 내용**
- 수강생 언리얼 엔진 설치 상태 점검 및 오류 해결
- Pawn과 Character 직접 조작 비교 체험:
  - Pawn: `AddActorWorldOffset`으로 단순 좌표 이동
  - Character: CMC에 의한 물리 기반 자연스러운 이동

## 실습 수업 전환의 배경

2주차 수업에서 강사는 전체 이론 위주의 진행에 대해 다음과 같이 인식을 밝혔다:

> **"이론 수업만으로는 흥미 유발이나 실제 체험 제공이 힘들다"**

이에 따라 3주차부터 실습 비중을 확대하였으며, 실습 수업으로 전환한 주요 동기는 다음과 같다:

1. **흥미 유발의 한계**: 이론만으로는 수강생의 지속적인 관심을 유지하기 어려움
2. **실제 체험 부재**: 직접 만들고 결과를 보는 경험 없이는 이해도와 동기 모두 떨어짐
3. **선행 이론 충족**: 1~2주차에 GameMode/GameState 등 핵심 이론을 마쳤으므로, 실습에 필요한 기초는 갖춰진 상태

3주차에 실제로 언리얼 엔진을 설치하고 Pawn vs Character 움직임을 직접 비교 체험하는 실습이 도입됨.
