## p.12 — 왜 PlayerController가 필요한가?

- **문제:** 캐릭터 내부에 입력 처리 코드 → 캐릭터가 죽으면 입력도 사라짐
- 탑다운·RTS처럼 폰을 교체하면 입력 코드도 함께 이사해야 하는 문제
- **해결:** 입력·카메라·UI를 `PlayerController`로 분리
  - 폰이 죽어도 Controller는 살아있음 → Respawn 구현 가능
  - `Possess()` 한 번으로 다른 폰 조종 가능

---

## p.13 — PlayerController 핵심 개념

- `APlayerController` extends `AController`
- 한 플레이어당 하나, 레벨 전환에도 유지
- 주요 함수: `Possess(APawn*)`, `UnPossess()`, `GetPawn()`
- 입력 처리: `SetupInputComponent` 또는 Enhanced Input 바인딩
- UI 관리: `CreateWidget + AddToViewport` 권장 위치

---

## p.14 — Possess 실행 흐름

```
GameMode::SpawnDefaultPawnFor()  →  Pawn 스폰
GameMode::RestartPlayer()        →  Possess 호출
APlayerController::Possess(pawn)
  └─ Pawn::PossessedBy(controller) 호출
       └─ OnPossess 이벤트 발생 (BP에서 오버라이드 가능)
이후 PlayerController가 입력을 해당 Pawn으로 전달
```

---

## p.15 — 핵심 규칙: One Controller, One Pawn

- PlayerController는 한 번에 하나의 Pawn만 Possess 가능
- 멀티플레이: 각 플레이어마다 별도 PlayerController 인스턴스
- AI: `AIController`가 같은 역할 수행 (PlayerController 미사용)
- Possess 교체 시 이전 Pawn은 자동 UnPossess
- 오늘 실습: `BP_ThirdPersonCharacter`를 그대로 사용

## p.18 — PlayerState 개념

- `APlayerState`: HP, 점수, 이름, 레벨, 경험치 등 플레이어 고유 데이터 저장
- `PlayerController` · `Pawn` 모두에서 접근 가능 (`GetPlayerState<AMyPlayerState>()`)
- Pawn이 파괴되어도 PlayerState는 살아있음
- 멀티플레이: 서버에서 Replicate → 모든 클라이언트에 동기화
- **오늘 목표:** BP_PlayerState에 HP 변수 추가 → 데미지 처리

---

## p.19 — GameState vs PlayerState

| 항목 | GameState | PlayerState |
|------|-----------|-------------|
| 저장 대상 | 게임 전체 데이터 (웨이브 번호, 남은 시간) | 개별 플레이어 데이터 (HP, 점수, 킬 수) |
| 공유 범위 | 모든 플레이어 공유 | 해당 플레이어 전용 |
| 관리 주체 | GameMode | PlayerController |
| 부모 클래스 | `AGameStateBase` | `APlayerState` |

## p.23 — Enhanced Input: 3가지 핵심 에셋

| 에셋 | 역할 | 예시 |
|------|------|------|
| `InputAction (IA_*)` | '무엇'을 하는지 | IA_Move, IA_Jump, IA_Sprint |
| `InputMappingContext (IMC_*)` | 키와 액션 매핑 | IMC_Default: WASD→IA_Move |
| `InputModifier` | 값 변환 | Negate, Swizzle YXZ, DeadZone |

> `Swizzle YXZ`: Y축 입력을 X로 전환 → 전후 이동에 사용

## p.24 — 런타임 IMC 추가/제거

```cpp
UEnhancedInputLocalPlayerSubsystem* Sub =
    ULocalPlayer::GetSubsystem<UEnhancedInputLocalPlayerSubsystem>(PlayerController);
Sub->AddMappingContext(IMC_Default, /*Priority=*/0);
// Priority가 높은 IMC가 같은 키를 먼저 처리
// 예: 탈것 탑승 시 IMC_Vehicle(Priority=1) 추가
Sub->RemoveMappingContext(IMC_Default); // 해제
```

## p.25 — IA 바인딩 패턴 (C++)

```cpp
// SetupPlayerInputComponent 내부
UEnhancedInputComponent* EIC = Cast<UEnhancedInputComponent>(InputComponent);

// Started: 키를 처음 누르는 순간 1회
EIC->BindAction(IA_Jump, ETriggerEvent::Started, this, &AMyChar::Jump);

// Triggered: 키를 누르는 동안 매 프레임
EIC->BindAction(IA_Move, ETriggerEvent::Triggered, this, &AMyChar::Move);

// Completed: 키를 뗄 때 1회
EIC->BindAction(IA_Sprint, ETriggerEvent::Completed, this, &AMyChar::StopSprint);
```

## p.39 — 실습 3 Step 2: TakeDamage 함수 구현

```
함수: TakeDamage(DamageAmount: Float)
  CurrentHP = Clamp(CurrentHP - DamageAmount, 0, MaxHP)

함수: IsAlive() → Boolean
  Return (CurrentHP > 0)
```

> ⚠️ Clamp 노드를 사용해 HP가 0 미만 / MaxHP 초과가 되지 않도록 처리
